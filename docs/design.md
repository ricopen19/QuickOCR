# QuickOCR 機能仕様書

## 1. プロジェクト概要

### 1.1 目的
macOS上で画面の任意の矩形範囲を選択し、そこに含まれるテキストをVision APIを使用して高速にOCR処理し、クリップボードにコピーするアプリケーションを開発する。

### 1.2 ターゲットユーザー
- 画面上のテキストを頻繁にコピーする必要があるユーザー
- PDFや画像からテキストを抽出したいユーザー
- キーボード操作を重視する効率的な作業を好むユーザー

## 2. 技術仕様

### 2.1 開発環境
- **言語**: Swift
- **UIフレームワーク**: SwiftUI
- **OCRエンジン**: Vision Framework (VNRecognizeTextRequest)
- **対象OS**: macOS 13 Ventura以降
- **最小デプロイメントターゲット**: macOS 13.0

### 2.2 使用するフレームワーク
- SwiftUI: UI構築
- Vision: テキスト認識
- AppKit: グローバルショートカット、スクリーンキャプチャ
- UserNotifications: 通知表示
- AVFoundation: 音声フィードバック（オプション）

### 2.3 パフォーマンス要件
- OCR処理時間: 矩形選択完了から1秒以内にクリップボードにコピー完了
- メモリ使用量: バックグラウンド時は50MB以下
- CPU使用率: OCR処理中以外は5%以下

## 3. 機能要件

### 3.1 コア機能

#### 3.1.1 アプリケーション起動
- **起動方式**: グローバルショートカットキー
- **デフォルトキーバインド**: `Cmd + Shift + O`
- **動作**:
  1. ショートカットキー押下
  2. 全画面が暗転（スクリーンショット風のオーバーレイ表示）
  3. 矩形選択モードに遷移

#### 3.1.2 矩形範囲選択UI
- **UIスタイル**: macOSネイティブスクリーンショット風
- **視覚効果**:
  - 画面全体を暗くするオーバーレイ（半透明の暗いレイヤー）
  - マウスドラッグで矩形範囲を選択
  - 選択中の矩形内は明るく表示（元の画面が見える）
  - 矩形の枠線は明確に表示（青または白のボーダー）
- **操作**:
  - マウスドラッグで矩形を描画
  - ドラッグ中は矩形のサイズをリアルタイムで表示
  - ドラッグを離すとOCR処理を開始
  - `Esc`キーで選択モードをキャンセル
  - マウス右クリックでキャンセル

#### 3.1.3 OCR処理
- **対応言語**: 日本語、英語
- **認識設定**:
  - 言語優先順位: 日本語 > 英語
  - 認識レベル: `accurate` (精度優先)
  - 自動言語検出: 有効
- **処理フロー**:
  1. 選択された矩形範囲をキャプチャ
  2. Vision Frameworkを使用してテキスト認識
  3. 認識されたテキストを取得
  4. テキストフォーマット処理を適用（設定により）
  5. クリップボードにコピー
  6. 通知を表示

#### 3.1.4 テキストフォーマット処理
- **基本処理**: 余分な改行を削除
- **詳細**:
  - 連続する改行（`\n\n`）は保持
  - 単一の改行で区切られたテキストは結合
  - PDF等から抽出した際の不自然な改行を除去
- **設定**: 設定画面でON/OFF切り替え可能
- **デフォルト**: 有効

#### 3.1.5 クリップボードへのコピー
- **動作**:
  - OCR処理完了後、自動的にシステムクリップボードにコピー
  - 既存のクリップボード内容は上書き
- **コピー形式**: プレーンテキスト（`NSPasteboard.PasteboardType.string`）

#### 3.1.6 通知表示
- **タイミング**: クリップボードへのコピー完了後
- **表示内容**:
  - タイトル: "OCR完了"
  - 本文: 抽出されたテキストの最初の3行または100文字（プレビュー）
  - サブタイトル: "クリップボードにコピーしました"
- **通知タイプ**: macOSシステム通知（UserNotifications）
- **通知の動作**:
  - 数秒後に自動的に消える
  - クリックしても特別なアクションは実行しない

#### 3.1.7 音声フィードバック（オプション機能）
- **タイミング**: OCR処理完了時
- **音声**: システム標準のサウンド（例: "Glass"）
- **設定**: 設定画面でON/OFF切り替え可能
- **デフォルト**: 無効

### 3.2 設定機能

#### 3.2.1 設定画面
- **アクセス方法**: メニューバーアイコン > "設定" または `Cmd + ,`
- **設定項目**:
  1. **ショートカットキー設定**
     - カスタムキーバインドの設定
     - キーコンビネーションのリアルタイムプレビュー
     - 他のアプリとの競合チェック

  2. **ログイン時自動起動**
     - チェックボックスでON/OFF
     - デフォルト: OFF

  3. **音声フィードバック**
     - チェックボックスでON/OFF
     - デフォルト: OFF

  4. **テキストフォーマット処理**
     - チェックボックスでON/OFF
     - デフォルト: ON

  5. **対応言語**
     - 日本語と英語（固定、変更不可）
     - 表示のみ

#### 3.2.2 メニューバーアイコン
- **表示位置**: macOSメニューバー右側
- **アイコンデザイン**: シンプルなOCRアイコン（テキストと矩形のシンボル）
- **メニュー内容**:
  - "OCRを実行" - ショートカットキー表示付き
  - "設定..."
  - "ログイン時に自動起動" - チェックマーク表示
  - セパレータ
  - "QuickOCRについて"
  - "終了"

### 3.3 エラーハンドリング

#### 3.3.1 権限エラー
- **シナリオ**: 画面収録権限が付与されていない
- **動作**:
  - アラートダイアログを表示
  - タイトル: "権限が必要です"
  - メッセージ: "QuickOCRは画面を読み取る権限が必要です。システム設定を開いて、プライバシー > 画面収録 でQuickOCRを許可してください。"
  - ボタン: "システム設定を開く", "キャンセル"
- **システム設定を開く**:
  - システム設定のプライバシー > 画面収録を直接開く

#### 3.3.2 OCR失敗
- **シナリオ**: テキストが認識できない、または選択範囲にテキストがない
- **動作**:
  - 通知を表示
  - タイトル: "OCR失敗"
  - メッセージ: "テキストを認識できませんでした。別の範囲を選択してください。"
- **クリップボード**: 変更なし

#### 3.3.3 システムエラー
- **シナリオ**: その他の予期しないエラー
- **動作**:
  - 通知を表示
  - タイトル: "エラーが発生しました"
  - メッセージ: "予期しないエラーが発生しました。アプリを再起動してください。"
- **ログ**: エラー詳細をコンソールに出力（開発用）

## 4. UI/UXフロー

### 4.1 基本的な使用フロー
```
1. [ユーザー] グローバルショートカットキー（Cmd+Shift+O）を押下
   ↓
2. [アプリ] 画面全体に暗いオーバーレイを表示
   ↓
3. [ユーザー] マウスをドラッグして矩形範囲を選択
   ↓
4. [アプリ] 選択範囲をハイライト表示
   ↓
5. [ユーザー] マウスボタンを離す
   ↓
6. [アプリ] OCR処理を実行（最大1秒）
   ↓
7. [アプリ] テキストをクリップボードにコピー
   ↓
8. [アプリ] 通知を表示（テキストプレビュー付き）
   ↓
9. [アプリ] オーバーレイを閉じて通常状態に戻る
   ↓
10. [ユーザー] 他のアプリでCmd+Vで貼り付け
```

### 4.2 キャンセルフロー
```
1. [ユーザー] グローバルショートカットキー（Cmd+Shift+O）を押下
   ↓
2. [アプリ] 画面全体に暗いオーバーレイを表示
   ↓
3. [ユーザー] Escキーを押下 または 右クリック
   ↓
4. [アプリ] オーバーレイを閉じて通常状態に戻る
```

### 4.3 初回起動フロー
```
1. [ユーザー] アプリを初めて起動
   ↓
2. [アプリ] ウェルカム画面を表示
   ├─ アプリの機能説明
   ├─ 必要な権限の説明
   └─ デフォルトショートカットキーの案内
   ↓
3. [アプリ] 権限リクエストダイアログを表示
   ↓
4. [ユーザー] 権限を許可
   ↓
5. [アプリ] メニューバーに常駐して準備完了
   ↓
6. [アプリ] 準備完了の通知を表示
```

## 5. データ設計

### 5.1 設定データ
設定は`UserDefaults`に保存

```swift
struct AppSettings {
    var globalShortcutKey: KeyboardShortcut = .default // Cmd+Shift+O
    var launchAtLogin: Bool = false
    var enableSoundFeedback: Bool = false
    var enableTextFormatting: Bool = true
    var supportedLanguages: [String] = ["ja", "en"] // 固定
}
```

### 5.2 OCR結果データ
```swift
struct OCRResult {
    var text: String
    var confidence: Float
    var recognizedLanguage: String
    var processingTime: TimeInterval
    var boundingBox: CGRect
    var timestamp: Date
}
```

## 6. 非機能要件

### 6.1 セキュリティとプライバシー
- **権限管理**:
  - 画面収録権限のみを要求
  - 権限の使用目的を明確に説明
- **データ保存**:
  - OCR結果をローカルに永続化しない
  - 履歴機能は実装しない（将来的に検討）
- **ネットワーク通信**:
  - 一切のネットワーク通信を行わない
  - すべての処理はローカルで完結

### 6.2 パフォーマンス
- **起動時間**: アプリ起動から使用可能まで1秒以内
- **メモリ使用量**:
  - バックグラウンド: 50MB以下
  - OCR処理中: 150MB以下
- **CPU使用率**:
  - アイドル時: 5%以下
  - OCR処理中: 80%以下（短時間のみ）

### 6.3 ユーザビリティ
- **レスポンス時間**:
  - UI操作への応答: 100ms以内
  - OCR処理: 1秒以内
- **エラーメッセージ**:
  - ユーザーが理解しやすい日本語のメッセージ
  - 問題解決の手順を明確に提示
- **アクセシビリティ**:
  - VoiceOver対応（将来的に検討）
  - 高コントラストモード対応

### 6.4 保守性
- **コード品質**:
  - Swift標準のコーディング規約に準拠
  - 適切なコメントとドキュメント
  - ユニットテストのカバレッジ70%以上
- **ログ**:
  - 開発ビルド: 詳細なデバッグログ
  - リリースビルド: エラーログのみ

## 7. テスト計画

### 7.1 ユニットテスト
- テキストフォーマット処理のロジック
- 設定の保存と読み込み
- OCR結果のパース処理

### 7.2 統合テスト
- OCR処理の全体フロー
- 権限チェックとエラーハンドリング
- 通知システムとの統合

### 7.3 E2Eテスト（手動）
- 実際の画面からのテキスト抽出
- 各種設定の変更と反映
- エラーシナリオの検証

### 7.4 テストケース例
1. **正常系**:
   - 日本語テキストのOCR
   - 英語テキストのOCR
   - 混在テキストのOCR
   - 複数行テキストのOCR

2. **異常系**:
   - テキストが存在しない画像
   - 非常に小さいフォント
   - 権限が付与されていない状態
   - ショートカットキーの競合

3. **境界値**:
   - 非常に大きい矩形範囲
   - 非常に小さい矩形範囲
   - 画面端の選択

## 9. 将来的な拡張機能（検討事項）

以下の機能は初期リリースには含めないが、将来的に検討する：

- **履歴機能**: 過去にOCRしたテキストの履歴を保存・検索
- **複数言語対応**: 中国語、韓国語などの追加
- **テキスト編集機能**: OCR結果を編集してからコピー
- **フォーマット保持**: テキストのフォント情報や色を保持
- **連続OCRモード**: 複数の矩形を連続で選択
- **クラウド同期**: 複数デバイス間での履歴同期
- **プラグインシステム**: サードパーティによる機能拡張
- **自動翻訳**: OCR後に自動で翻訳
